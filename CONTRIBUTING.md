# Contributing to Software Archeology

## Welcome
Thank you for contributing. This repository is built for enterprise modernization practitioners, AI engineers, and open-source collaborators.

## Contribution Process
- Fork the repository.
- Create a branch with a descriptive name.
- Add or improve specification artifacts in Markdown.
- Use consistent section structure and professional language.
- Open a pull request with a clear summary and review checklist.

## Content Standards
- Use deterministic, model-friendly wording.
- Keep each artifact objective and scoped.
- Avoid vague or marketing-first language.
- Reference examples when adding new specifications.

## Review Guidelines
- Confirm each file includes title, objective, scope, instructions, expected outputs, example outputs, analysis rules, and suggested extensions.
- Check that prompts and templates are reusable and specific.
- Validate that new content supports enterprise modernization use cases.

## Community Expectations
- Be respectful, constructive, and inclusive.
- Report issues or gaps using GitHub issues.
- Tag maintainers for complex changes.
