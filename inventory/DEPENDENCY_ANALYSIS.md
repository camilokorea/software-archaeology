# Dependency Analysis

## Objective
Inventory third-party libraries, frameworks, and runtime dependencies.

## Scope
This artifact covers dependency analysis.md.

## Instructions
Review dependency manifests, lockfiles, and platform-specific packages.

## Expected Outputs
A prioritized list of dependencies, versions, and modernization risk flags.

## Example Outputs
A dependency matrix that identifies obsolete, insecure, or unsupported packages.

## Analysis Rules
Use automated dependency scanning tools to verify the list.

## Suggested Extensions
Add license risk classification and dependency aging metrics.
